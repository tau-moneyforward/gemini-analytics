# you asked

API 連携時のリダイレクト URI って何？

---

# gemini response

API連携時のリダイレクトURI（リダイレクト・ユーアールアイ）とは、**ユーザーが認証・認可を完了した後に、元のアプリケーションに戻ってくるための「戻り先URL」**のことです。

特に、OAuth 2.0のような認証フローで使われる、非常に重要なセキュリティ設定です。

---

## 🧐 どのような仕組み？

API連携（例えば「Googleアカウントでログイン」機能）の具体的な流れで見てみましょう。

1.  **ユーザーの操作:**
    あなたのWebサービス（アプリA）で、ユーザーが「Googleでログイン」ボタンを押します。
2.  **認証サーバーへ移動:**
    ユーザーはGoogleのログイン画面（認証サーバー）に飛ばされます。
3.  **認証と認可:**
    ユーザーはGoogleにログインし、「アプリAにあなたのプロフィール情報へのアクセスを許可しますか？」という画面で「はい」を押します。
4.  **リダイレクト（戻る）:**
    認証が成功すると、Googleはユーザーを「アプリA」に戻そうとします。このとき、**Googleはあらかじめ指定されていた「リダイレクトURI」**（例: `https://app-a.com/callback`）に、ユーザーを自動的に転送（リダイレクト）させます。
5.  **コードの受け渡し:**
    その際、GoogleはリダイレクトURIのURLパラメータとして、**「認証コード」（一時的なカギ）**を付け加えます。
    （例: `https://app-a.com/callback?code=XXXXXXX`）
6.  **連携完了:**
    アプリAは、戻ってきたユーザーを`callback`ページで受け止め、URLに付いてきた認証コードを使ってGoogleと通信し、正式なAPIキー（アクセストークン）を取得してログインを完了させます。

---

## 🔒 なぜ重要？（セキュリティ）

リダイレクトURIが重要な最大の理由は、**セキュリティの確保**（なりすまし防止）です。

もし、この「戻り先URL」を誰でも自由に設定できてしまうと、悪意のある攻撃者が自分のサイトのURLを指定して認証を行わせ、ユーザーの認証コードを盗み取ることができてしまいます。（これは「リダイレクト攻撃」と呼ばれます）

これを防ぐため、APIを提供する側（Googleなど）は、開発者がアプリケーションを登録する際に、**「認証コードを送り返しても良い、安全なURL（＝リダイレクトURI）」を事前に厳密に登録（ホワイトリスト登録）**することを義務付けています。

> **まとめ:**
> リダイレクトURIは、API連携における**「認証OKの証拠（認証コード）を受け取るための、事前に登録された安全な窓口URL」**です。

---
