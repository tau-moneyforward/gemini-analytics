# you asked

JSON のリストって順番が保証されてる？

---

# gemini response

はい、JSONのリスト（正しくは「配列」や「Array」と呼び、`[ ]` で囲まれた部分）は、仕様上 **順序が保証されています。**

データの書き出し順と、読み込み順は必ず一致します。

混同しやすい「オブジェクト」との違いを含めて、詳しく解説します。

-----

### 1\. 配列（リスト） `[ ... ]`

**→ 順序は保証されます**

JSONの仕様（RFC 8259）において、配列は「値の**順序付き**シーケンス（ordered sequence）」と定義されています。

```json
{
  "users": ["Aさん", "Bさん", "Cさん"]
}
```

この場合、プログラムで読み込んだ際も必ず **Aさん → Bさん → Cさん** の順番になります。インデックス（0番目、1番目...）も維持されます。

### 2\. オブジェクト（連想配列） `{ ... }`

**→ 順序は保証されません（仕様上）**

キーと値のペアであるオブジェクトは、仕様上「**順序なし**（unordered）」とされています。

```json
{
  "id": 1,
  "name": "Tanaka",
  "age": 25
}
```

最近の多くのプログラミング言語やブラウザの実装では、便宜上「書いた順」で扱われることが多いですが、**仕様としては順序がバラバラになっても文句は言えません。**
（例：「age」→「id」→「name」の順で読み込まれる可能性があります）

-----

### まとめ

| 種類 | 記号 | 順序の保証 | 備考 |
| :--- | :---: | :---: | :--- |
| **配列 (List/Array)** | `[ ]` | **あり ✅** | リストの中身は順番通りに処理されます。 |
| **オブジェクト (Object)** | `{ }` | **なし ❌** | 順番を期待するロジックを書いてはいけません。 |

> **注意点**
> 配列の順序はJSONの仕様として保証されていますが、それを受け取る側のプログラム（古いライブラリや特定のデータベースなど）によっては、保存時に順序が変わってしまうケースが稀にあります。ただし、通常のJSONパース処理であれば心配する必要はありません。

-----

順番を確実に守りたいデータ構造を作る場合は、必ず **`[ ]`（配列）** を使用してください。

**特定のプログラミング言語（JavaScript, Python, Goなど）でのJSONの扱い方や、データの並び替え（ソート）方法について知りたい場合は、続けてご質問ください。**
