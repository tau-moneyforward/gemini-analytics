# you asked

うまく言ってないね。どこを見れば問題を特定できる？

---

# gemini response

うまくいかない時は、以下の**4つのポイント**を確認すると、原因が特定できるはずです。特に「実行数（ログ）」と「アカウントの状態」がよくある原因です。

### 1. エディタの「実行数」ログを確認する（ここが一番重要）

スクリプトが「なぜ動かなかったのか」の証拠はここに残ります。

1.  GASのエディタ画面に戻ります。
2.  左側のメニューから **「実行数」**（再生マークのようなアイコン ▶ に時計がついたもの）をクリックします。
3.  右側にリストが表示されます。
    * **「失敗」と出ている場合:** その行をクリックすると、詳細なエラーメッセージ（「ReferenceError」や「権限がありません」など）が表示されます。
    * **何も表示されていない場合:** そもそもスクリプトが起動していません。手順2以降へ進んでください。

---

### 2. エディタから手動で「実行」して、権限を通す

「テストデプロイ」でインストールしただけでは、**認証（権限許可）のポップアップが出るタイミングがなく、裏でエラーになって止まっている**ことがよくあります。

これを解消するために、一度手動で動かします。

1.  エディタの上部にあるプルダウンメニュー（関数名を選ぶところ）で **`onOpen`** を選びます。
2.  **「実行」** ボタンを押します。
3.  **「承認が必要です」** というポップアップが出たら、許可を進めてください。
4.  実行ログに「完了」と出たら、ドキュメントのタブに戻ってリロードしてみてください。

---

### 3. メニューの表示場所を確認する

スタンドアロン（アドオン）として入れた場合、メニューが出る場所が変わります。

* **以前（コンテナバインド）:** メニューバーに「画像ツール」と直接表示されていました。
* **今回（アドオン）:** メニューバーの **「拡張機能」** の中にある **「画像抽出ツール」**（またはマニフェストで設定した名前）の中に表示されます。

「拡張機能」をクリックしても名前が出てこない場合は、インストール自体がうまく認識されていません。

---

### 4. 「複数のGoogleアカウント」でログインしていないか確認

これが**最も厄介で、最も多い原因**です。

もし、Chromeブラウザで「個人のGmail」と「会社のGoogle Workspace」など、**複数のGoogleアカウントに同時にログインしている**場合、GASのアドオン（特にテストデプロイ）は**高確率で動作しません**（どのアカウントで実行すべきかブラウザが迷って、権限エラーを吐かずに沈黙します）。

**確認・解決策:**
1.  Chromeの「シークレットウィンドウ」を開きます。
2.  スクリプトを作成した **1つのGoogleアカウントだけ** でログインします。
3.  Googleドキュメントを開き、「拡張機能」メニューを確認します。

もしシークレットウィンドウで表示されたなら、原因は「複数アカウントログイン」です。

---

**まずは「2. エディタから手動実行」を試してみてください。**
これでエラーが出るか、承認画面が出れば、解決に近づきます。どうなりましたか？

# you asked

実行したら、ログにこんなのがでた

Dec 8, 2025, 10:03:55 AMErrorException: Cannot call DocumentApp.getUi() from this context.
    at onOpen(Code:9:15)

---

# gemini response

そのエラーが出るのは\*\*「正常」**です。そして、そのエラーが出たことによって**「なぜメニューが表示されないのか」の原因がわかりました。\*\*

### 原因：認証（許可）がまだ終わっていない

`Cannot call DocumentApp.getUi() from this context.` というエラーは、「スクリプトエディタから直接実行しているため、操作対象となる『ドキュメントの画面（UI）』が存在しない」という意味です。

しかし、一番の問題は\*\*「エラーで止まってしまったため、権限の承認ポップアップが表示されず、スクリプトがまだ許可されていない」\*\*ことです。
許可されていないスクリプトは、ドキュメントを開いてもブロックされて動かないため、メニューが表示されません。

-----

### 解決策：強制的に「承認画面」を出す

UI（画面）を使わない、無害な関数を一つ作って実行し、そこで権限の許可を通します。

1.  GASのエディタで、以下の短いコードを一番下に追加してください。

<!-- end list -->

```javascript
// 権限の承認画面を出すためだけのダミー関数
function forceAuth() {
  // 新規ドキュメントを一瞬だけ作成して、すぐ閉じる（これで権限を要求させる）
  DocumentApp.create('Auth Check Temp Doc');
  console.log('認証完了！この関数はもう不要です。');
}
```

2.  上のメニューで実行する関数を **`forceAuth`** に切り替えます。
3.  **「実行」** を押します。
4.  今度こそ **「承認が必要です」** というポップアップが出るはずです。
      * 「権限を確認」→ アカウント選択 → （詳細）→「（安全ではないページ）に移動」→「許可」と進んでください。
5.  ログに「認証完了！」と出れば成功です（Googleドライブに空のドキュメントが1つ作られますが、後で消してOKです）。

-----

### 確認手順

認証が済んだら、もう一度ドキュメント側を確認します。

1.  テストしたい Googleドキュメント のタブに戻ります。
2.  ページを **リロード** します。
3.  数秒待ってから、**「拡張機能」** メニューをクリックします。
4.  **「画像抽出ツール」**（または設定した名前）が表示されているか確認してください。

これで表示されるはずです！

# gemini response

申し訳ありません！これは私のミスです。

直前に追加していただいた `forceAuth` 関数（新しいドキュメントを作る命令）には、**「ドキュメント全体へのアクセス権限（`https://www.googleapis.com/auth/documents`）」** が必要なのですが、マニフェスト（`appsscript.json`）には\*\*「今開いているドキュメントだけ（`.currentonly`）」\*\* という、より厳しい制限をかけてしまっていました。そのため「権限が足りない」と怒られています。

マニフェストファイルを修正して、制限を少し緩めれば解決します。

### 手順：マニフェストの修正

1.  GASエディタの左側で **`appsscript.json`** を開きます。
2.  `oauthScopes` の部分を、以下のように書き換えてください（`.currentonly` を削除します）。

**変更前（今の状態）:**

```json
"https://www.googleapis.com/auth/documents.currentonly",
```

**変更後（これに書き換える）:**

```json
"https://www.googleapis.com/auth/documents",
```

**全体の `appsscript.json` はこうなります:**

```json
{
  "timeZone": "Asia/Tokyo",
  "dependencies": {},
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8",
  "oauthScopes": [
    "https://www.googleapis.com/auth/documents",
    "https://www.googleapis.com/auth/script.container.ui"
  ],
  "addOns": {
    "common": {
      "name": "画像抽出ツール",
      "logoUrl": "https://www.gstatic.com/images/branding/product/1x/docs_2020q4_48dp.png",
      "layoutProperties": {
        "primaryColor": "#4285F4"
      }
    },
    "docs": {
      "onFileScopeGrantedTrigger": {
        "runFunction": "onOpen"
      }
    }
  }
}
```

### 再実行

1.  書き換えたら **保存（💾）** します。
2.  もう一度、関数 **`forceAuth`** を選択して **「実行」** ボタンを押してください。
3.  今度こそエラーではなく **「承認が必要です」** というポップアップが出るはずです。
4.  許可を進めてください。

ログに「認証完了！」と出たら、ドキュメントをリロードして「拡張機能」メニューを確認してください。今度こそ表示されるはずです！

# you asked

Add-ons のメニューからインストールできるようにできない？

---

# gemini response

アドオンのメニュー（Google Workspace Marketplace）から「インストール」ボタンで導入できるようにするには、**「Google Workspace Marketplace への公開」** という手続きが必要です。

これは可能ですが、お使いの Google アカウントの種類（**個人のGmail** か **企業のGoogle Workspace** か）によって難易度が天と地ほど違います。

状況に合わせて、現実的なルートを説明します。

---

### パターンA：企業の Google Workspace アカウントの場合
（例: `yourname@company.com`）

**→ 可能です。しかも「社内限定公開」なら審査なしで即公開できます。**

これを行うと、社内の他のメンバーも Marketplace からあなたのツールをインストールできるようになります。

**手順の概要:**
1.  **Google Cloud Platform (GCP) プロジェクト** を作成し、GASと紐付ける（※課金アカウントの紐付けが必要な場合があります）。
2.  GCP側で **Google Workspace Marketplace SDK** API を有効にする。
3.  SDKの設定で「公開範囲」を **「非公開（このドメインのみ）」** に設定する。
4.  「ストアの掲載情報」（アイコンや説明文）を入力して公開ボタンを押す。

これだけで、社内のドキュメントのアドオンメニューから検索・インストールできるようになります。

---

### パターンB：個人の Gmail アカウントの場合
（例: `yourname@gmail.com`）

**→ 正直、おすすめしません（難易度が非常に高いです）。**

個人の場合、「自分専用に Marketplace に公開する（非公開設定）」という機能が使えません。公開＝**「世界中への一般公開」** になってしまいます。

**一般公開の壁:**
1.  **Googleの厳格な審査:** 英語でのやり取り、プライバシーポリシーの作成、動画での機能説明などが必要で、通過に数週間〜数ヶ月かかります。
2.  **コスト:** アプリの検証（OAuth Verification）に費用がかかる場合があります。

**結論:** 個人利用であれば、先ほどの **「デプロイをテスト」→「インストール」** が、Googleが用意している唯一の現実的な「自分専用インストール」の方法です。

---

### 「テストデプロイ」が消えてしまう問題の解決策

もし「テストデプロイでインストールしたはずなのに、しばらくするとメニューから消えている」ということであれば、それは **「ヘッドデプロイ（最新コード）」** をインストールしているため、セッションが切れると消える仕様だからかもしれません。

個人利用で「消えないように」インストールを維持するコツは以下の通りです。

1.  **バージョンを作成してデプロイする:**
    * エディタ右上の「デプロイ」> **「新しいデプロイ」** をクリック。
    * 種類の選択（歯車）から **「アドオン」** を選択。
    * 説明に「v1」などと入れて **「デプロイ」** をクリック。
    * **「デプロイID」** をコピーしておきます。

2.  **そのバージョンをインストールする:**
    * 「デプロイ」> **「デプロイをテスト」** をクリック。
    * **重要:** 左上のプルダウンが「Head（最新）」になっているのを、**「Version 1（先ほど作ったバージョン）」** に切り替えます。
    * その状態で **「インストール」** をクリックします。

これで、「開発中の不安定なコード」ではなく「固定されたバージョンのアプリ」としてブラウザに認識されるため、比較的安定してメニューに残り続けるはずです。

**今の環境は、個人アカウント（Gmail）でしょうか？それとも組織アカウントでしょうか？**
それによって、詳しいGCPの設定手順をご案内するかどうかが変わります。
