# gemini response

**DITA**（ディータ）とは、**Darwin Information Typing Architecture** の略称で、技術ドキュメント（マニュアルや仕様書）を作成・管理するための、**XMLベースの国際標準規格**（OASIS規格）です。

一言で言うと、**「ドキュメントを『章』や『ページ』で書くのではなく、『情報の部品（トピック）』として書き、それをレゴブロックのように組み合わせてマニュアルを作る仕組み」**のことです。

製造業、ソフトウェア開発、医療機器メーカーなど、大規模で複雑なドキュメントを扱う現場で広く使われています。

---

### 1. DITAの最大の特徴：トピック指向
従来のWordのような執筆方法と決定的に違うのが、**「トピック指向（Topic-based Writing）」**という考え方です。

* **従来:** 第1章から順番に、長い文章を書いていく。
* **DITA:** 情報を**「トピック」**という小さな単位（部品）に切り分けて書く。

#### DITAにおける3つの基本トピック型
DITAでは、書く内容によって「型」が決まっています。

1.  **Concept（概念）**
    * **「それは何か？」**を説明する。
    * 例：「Adminaとは何か」「SSOの概要」
2.  **Task（タスク）**
    * **「どうやるのか？」**（手順）を説明する。
    * 例：「ユーザーを追加する手順」「パスワードをリセットする方法」
3.  **Reference（参照）**
    * **「仕様・データ」**を説明する。
    * 例：エラーコード一覧、パラメータ表、API仕様

### 2. DITAマップ（構成図）
バラバラに書いた「トピック」を、**DITAマップ**という設計図（目次のようなもの）で繋ぎ合わせます。

* **マップA（初心者向けガイド）** = 「概要トピック」＋「基本操作トピック」
* **マップB（管理者マニュアル）** = 「概要トピック」＋「詳細設定トピック」＋「エラーコード表」

このように、**「概要トピック」という一つの部品を、複数のマニュアルで使い回す（再利用する）**ことができます。

---

### 3. DITAを導入するメリット

大規模なドキュメント制作において、以下のような強力なメリットがあります。

* **コンテンツの再利用 (Content Reuse)**
    * 同じ注意書きや手順をコピペする必要がありません。一箇所修正すれば、それを参照している全マニュアルに修正が反映されます。
* **マルチチャネル出版**
    * 一つのDITAソース（XML）から、PDF、HTML、Webヘルプ、EPUBなどを自動生成できます（DITA Open Toolkitというツールを使います）。
* **翻訳コストの削減**
    * 変更があった「トピック（部品）」だけを翻訳に出せば良いため、翻訳料金を大幅に下げられます。
* **構造の強制**
    * XMLタグで「ここは手順」「ここは結果」と厳密に決まっているため、誰が書いても文書構造が統一されます。

### 4. デメリット・課題

* **学習コストが高い:** XMLの知識や、独特のライティングルールを覚える必要があります。
* **導入コストが高い:** 執筆には専用のエディタ（Oxygen XML Editorなど）や、管理システム（CCMS）が必要になることが多く、高額になりがちです。
* **小規模には不向き:** 10ページ程度のメモ書きや、頻繁に構造が変わるブログ記事などには、Markdownなどのほうが向いています。

### まとめ

| 項目 | 特徴 |
| :--- | :--- |
| **名前** | Darwin Information Typing Architecture |
| **形式** | XML (構造化データ) |
| **思想** | 部品化して使い回す（トピック指向） |
| **向いているもの** | 製品マニュアル、API仕様書、法律・規約集 |
| **向いていないもの** | 小説、エッセイ、ブログ、短い議事録 |

**「ドキュメントの管理・翻訳コストを下げたい」「マニュアルの表記揺れをなくしたい」**という管理者視点で注目されることが多い技術です。
