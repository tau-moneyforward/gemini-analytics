# you asked

「事業所 a1 と a2 と a3 は、事業所 a1 で社保手続き」「事業所 b1 と b2 は 事業所 b1 で社保手続き」といった構成のとき、この "構成" を何と呼ぶ？構成？体制？社保手続きの対応関係？

---

# gemini response

この「まとまり（グルーピング）のあり方」を指す言葉は、文脈（法律・システム設計・業務）によっていくつか候補がありますが、最も一般的で誤解がないのは以下の用語です。

### 1. 公的・法的な文脈（最も正式）
**「一括適用の単位（いっかつてきようのたんい）」**

行政（年金事務所）の視点では、まとまり一つひとつを「適用単位」と捉えます。
「関東と関西で**一括適用の単位**を分けています」と言えば、プロ同士の会話として最もスムーズに通じます。

### 2. システム設計・DB設計の文脈（エンジニア向け）
**「事業所親子関係（じぎょうしょ・おやこかんけい）」**

データ構造としては、a1を「親」、a2・a3を「子」とするツリー構造になるため、この言葉が最も直感的です。
仕様書などでは以下のように使い分けます。

* **構成そのもの:** 「事業所親子関係」「事業所階層」
* **グループ:** 「親子グループ」

### 3. UIや業務フローの文脈（ユーザー向け）
**「統括関係（とうかつかんけい）」** または **「申請ルート」**

社会保険の実務では、まとめ役の事業所を「統括事業所（とうかつじぎょうしょ）」、まとめられる側を「被一括事業所（ひいっかつじぎょうしょ）」と呼びます。
そのため、この繋がりを「統括関係」と呼ぶのが専門的かつ正確です。

### おすすめの使い分け（用例）

文脈に合わせて以下のように使い分けるのが「プロらしい」表現です。

| 文脈 | 推奨用語 | 例文 |
| :--- | :--- | :--- |
| **仕様書・定義** | **一括適用の単位** | 「関東と関西で、**一括適用の単位**を分ける設計とする。」 |
| **DB・ロジック** | **事業所親子関係** | 「**事業所親子関係**テーブルで、a1に紐づくa2, a3を定義する。」 |
| **画面・設定** | **統括設定 / 統括関係** | 「各事業所の**統括関係**を設定してください。」 |
| **ラフな会話** | **社保の管轄** | 「a2とa3は、a1の**管轄**に入ります。」 |

システム内の名称（ラベル）として使うなら、**「一括適用グループ」** や **「統括設定」** がユーザーにとって分かりやすいでしょう。
