# gemini response

FFmpegを使用して動画をMP4に変換するための、よく使われるコマンドパターンをまとめました。用途に合わせて使い分けてください。

-----

### 1\. 基本の変換（一番シンプル）

細かい設定を気にせず、とりあえずMP4にしたい場合はこれを入力してください。FFmpegが自動的に適切な設定を選んでくれます。

```bash
ffmpeg -i 入力ファイル名.mov 出力ファイル名.mp4
```

> **ポイント:** 入力ファイルの拡張子は `.mov`, `.avi`, `.webm`, `.mkv` など何でも大丈夫です。

-----

### 2\. 画質と互換性を重視（推奨）

H.264コーデックを指定して、スマホやPCなど**どこでも再生できるようにする**標準的なコマンドです。

```bash
ffmpeg -i input.mov -c:v libx264 -c:a aac output.mp4
```

  * `input.mov`: 元の動画
  * `-c:v libx264`: 映像をH.264（最も一般的な形式）にする
  * `-c:a aac`: 音声をAACにする
  * `output.mp4`: 完成した動画

-----

### 3\. 画質を調整する（ファイルサイズ削減）

画質とファイルサイズのバランスを調整したい場合は、`-crf` オプションを使います。

```bash
ffmpeg -i input.mov -c:v libx264 -crf 23 output.mp4
```

  * **-crf の数値:** 画質レベルを指定します（範囲は0〜51）。
      * **18:** ほぼ無劣化（ファイルサイズ大）
      * **23:** 標準（デフォルト・バランスが良い）
      * **28:** 画質を少し落としてファイルサイズを小さくする

-----

### 4\. 爆速変換（劣化なし・エンコードなし）

元の動画の中身（コーデック）が既にMP4と互換性がある場合（例: MKVコンテナに入っているが中身はH.264など）、再エンコードせずに「箱」だけを入れ替えることができます。**画質劣化ゼロで、数秒で終わります。**

```bash
ffmpeg -i input.mkv -c copy output.mp4
```

  * `-c copy`: 映像も音声もそのままコピーする（変換処理をしない）。

-----

### 5\. 解像度を変更してMP4にする（リサイズ）

例えば、動画をHD（1280幅）に縮小したい場合などです。

```bash
ffmpeg -i input.mov -vf scale=1280:-1 output.mp4
```

  * `scale=1280:-1`: 横幅を1280pxにし、高さは比率を維持して自動計算（`-1`）する。

-----

### オプション解説まとめ

| オプション | 意味 | 解説 |
| :--- | :--- | :--- |
| **-i** | input | 入力ファイルを指定します。 |
| **-c:v** | codec video | 映像の圧縮形式（`libx264`推奨）。`-vcodec` と書くこともあります。 |
| **-c:a** | codec audio | 音声の圧縮形式（`aac`推奨）。`-acodec` と書くこともあります。 |
| **-crf** | constant rate factor | 画質係数。数値が低いほど高画質・大容量になります。 |
| **-vf** | video filter | リサイズやクロップなどのフィルタを適用します。 |
